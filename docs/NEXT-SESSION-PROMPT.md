# Next Session Prompt — Smart Claude Memory

Paste the block below as your first message in any new session that uses
`smart-claude-memory`. It is the canonical "Golden Startup Prompt" plus the
v1.1.4 Architecture Guard additions, kept as a single copy-paste artefact so
session-end handovers stay deterministic.

```text
SYSTEM INITIALIZATION:
I am using the `smart-claude-memory` plugin. Follow these standards:

1. Check Readiness: run `init_project` to verify the workspace and health.
2. Sync State: run `sync_local_memory()` so the vector database is current
   with my local notes.
3. Operate via Tools: do NOT read large `.md` files directly — use
   `search_memory()` for context retrieval, and respect `md-policy.py`
   for all writes.
4. Format Consistency: keep new content aligned with the existing README
   style. After updates, run `sync_artefacts` to keep cloud + local in
   parallel.
5. MANDATORY DELEGATION: any read-heavy investigation touching > 3 files
   OR > 100 lines of raw output (Grep / Read / logs) MUST go through
   `delegate_task`. Request only the 2-paragraph synthesis.
6. Initial Sync (Core 3 Audit): `init_project` returns a `core3` block
   covering CLAUDE.md, README.md, and ARCHITECTURE.md. If any is missing
   or `core3.in_sync` is false, immediately delegate a Core-3 audit via
   `delegate_task` BEFORE any other work.
7. Modular Diagramming: Mermaid diagrams in ARCHITECTURE.md / README.md
   MUST be split into one block per `##` subsystem (≤ ~40 nodes each).
   GitHub silently fails to render oversized Mermaid graphs.
8. Session-End Lock: before ending the session, call
   `manage_backlog({ action: "session_end" })` to flush the backlog,
   regenerate per-section diagrams, and run `sync_artefacts`. Then post
   the `next_session_command_markdown` block from that response verbatim
   into chat as your final message — that is the user's copy-paste
   command for the next session.

First actions for THIS session:
- init_project()
- search_memory({ query: "Active Backlog", project_id: "<this-project-id>", k: 10 })
```

## How this file is consumed

- Generated by `manage_backlog({ action: "session_end" })` and referenced
  in the `next_session_command_markdown` field of that response.
- The agent posts the next-session command verbatim at session close.
- Users update this file when they want to change the boot ritual for
  every project that uses `smart-claude-memory`.

## When to update

- A new architectural rule is added to the Golden Startup Prompt in
  `README.md`. Mirror it here.
- The `init_project` response shape changes in a way that affects what
  the agent should do at boot.
- A new MCP tool replaces an existing one in the boot sequence.

Keep this file minimal. The boot prompt is its only job.
